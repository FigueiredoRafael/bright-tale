Create Products & Prices

Stripe Dashboard

Go to Stripe Dashboard → Products → + Add product. Create each product with the prices below. Copy every price_... ID.

ⓘ Free tier has no Stripe product — it's the default state (1,000 credits, $0). Only create the 3 paid plans + 3 addon packs.

Subscription Plans (recurring)

ProductMonthlyAnnualCredits / mo
BrightTale Starter $9/mo $84/yr ($7/mo) 5,000
BrightTale Creator $29/mo $276/yr ($23/mo) 15,000
BrightTale Pro $99/mo $948/yr ($79/mo) 50,000

Each product gets 2 prices (Monthly recurring + Yearly recurring). That's 6 price IDs.

Add-on Credit Packs (one-time)

ProductPriceCredits
1K Credit Pack $5 one-time 1,000
5K Credit Pack $20 one-time 5,000
15K Credit Pack $50 one-time 15,000

Each pack gets 1 price (one-time, mode payment). That's 3 price IDs.

  • Starter product created — monthly + annual prices copied
  • Creator product created — monthly + annual prices copied
  • Pro product created — monthly + annual prices copied
  • 1K Pack product + price created
  • 5K Pack product + price created
  • 15K Pack product + price created

Set Up Webhook

Stripe Dashboard

Go to Developers → Webhooks → + Add endpoint

Endpoint URL 
https://api.brighttale.io/api/billing/webhook

Events to subscribe:

Webhook Events 
checkout.session.completed
customer.subscription.created
customer.subscription.updated
customer.subscription.deleted
invoice.payment_succeeded
invoice.paid
⚠ After creating the endpoint, copy the Signing Secret (whsec_...). You'll need it for the env vars.
  • Webhook endpoint created with all 6 events
  • Signing secret (whsec_...) copied

Customer Portal

Stripe Dashboard

Go to Settings → Billing → Customer portal. Enable:

  • Cancel subscription
  • Update payment method
  • Switch plans (optional — lets users upgrade/downgrade from portal)
  • Save configuration

Environment Variables

11 Variables

Add to apps/api/.env.local (local dev) and Vercel project settings (production).

apps/api/.env.local 
# ── Stripe Core ──────────────────────────────
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

# ── Subscription Prices (6) ──────────────────
STRIPE_PRICE_STARTER_MONTHLY=price_...
STRIPE_PRICE_STARTER_ANNUAL=price_...
STRIPE_PRICE_CREATOR_MONTHLY=price_...
STRIPE_PRICE_CREATOR_ANNUAL=price_...
STRIPE_PRICE_PRO_MONTHLY=price_...
STRIPE_PRICE_PRO_ANNUAL=price_...

# ── Add-on Packs (3) ─────────────────────────
STRIPE_PRICE_ADDON_1K=price_...
STRIPE_PRICE_ADDON_5K=price_...
STRIPE_PRICE_ADDON_15K=price_...
⚠ Use sk_test_... keys for dev, sk_live_... for production. Never commit secret keys to git.
  • STRIPE_SECRET_KEY set
  • STRIPE_WEBHOOK_SECRET set
  • All 6 subscription price IDs set
  • All 3 addon pack price IDs set
  • Same variables set in Vercel (production)

Verify Setup

Restart the API, then run this curl to confirm all price IDs resolved:

Verify Plans 
curl -s -H "X-Internal-Key: $INTERNAL_API_KEY" \
  http://localhost:3001/api/billing/plans | jq .

Expected: each plan's stripePriceId fields should show actual price_... values, not null.

Test Webhook (Stripe CLI) 
stripe listen --forward-to localhost:3001/api/billing/webhook
stripe trigger checkout.session.completed

The listen command forwards Stripe events to your local API. Trigger a test event and confirm it processes without errors in the API logs.

  • /billing/plans returns non-null price IDs
  • Stripe CLI webhook forwarding works
  • Test checkout flow completes end-to-end
