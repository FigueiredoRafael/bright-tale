feat/persona-eeat-layer
25 commits mergeado → staging

Branch de longa duração (iniciada para a camada E-E-A-T de Personas) que culminou com a maior refatoração de infraestrutura WP do projeto: migração de schema, inversão de FK, remoção de endpoints legados, e redesign completo da UX de configuração WordPress.

Schema: FK invertida

Antes: channels.wordpress_config_id → wordpress_configs.id (N:1, config compartilhada).
Depois: wordpress_configs.channel_id → channels.id (1:1 com unique index, cascade delete). user_id e org_id removidos da tabela — redundantes com channel_id.

API: 6 endpoints globais → 5 channel-scoped

Deletados: POST/GET/GET(:id)/PUT/PATCH/DELETE /api/wordpress/config.
Adicionados: GET/POST/PUT/DELETE/test /api/channels/:id/wordpress com verificação de org ownership em cada rota.

Pipeline: sem configId

publishDraftSchema removeu configId. Os handlers agora resolvem via cadeia: body.channelId → draft.channel_id → project.channel_id. Drafts e projetos legados publicam sem migração de dados.

UI: /settings/wordpress deletada

745 linhas removidas. Nav, breadcrumb, settings card e chaves i18n limpos. Novo: WordPress card no Blog tab do canal com estados de connect/edit/test/remove e AlertDialog de confirmação.

Segurança corrigida

A rota POST /api/personas/:id/integrations/wordpress não validava que body.channelId pertencia à org do requester — qualquer usuário autenticado podia acessar credenciais WP de outro tenant. Corrigido com lookup de org_membership antes de acessar wordpress_configs.

Itens em aberto

O que ficou de fora do escopo hoje e pode ser endereçado a seguir.

Testes automatizados para rotas channel-scoped

A spec previa apps/api/src/routes/__tests__/channels.wordpress.test.ts com 5 casos (create, conflict, PUT sem password, delete→GET, cross-org). Não foi implementado — apenas testes Category C (DB-hitting) são viáveis; ficam como TODO-test.

URL-driven tab no Channel Detail

O link "Configure it here" aponta para /channels genérico porque a página de canal não suporta ?tab=blog via URL. Adicionar roteamento por query param seria a UX ideal para chegar direto no WordPress card.

Projetos legados sem channel_id

O picker de canal foi adicionado, mas projetos existentes precisam ser vinculados manualmente. Uma migration de backfill automático (derivar canal do idea.channel_id) reduziria o atrito.

apps/web — affiliate-portal types

Erros de typecheck pré-existentes em @tn-figueiredo/affiliate-portal continuam bloqueando npm run typecheck global. Não relacionado a este branch; requer atenção separada.
