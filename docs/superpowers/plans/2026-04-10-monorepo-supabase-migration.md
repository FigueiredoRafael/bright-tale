# Monorepo + Supabase Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure `bright-tale` from a single Next.js app (`bright-curios-workflow/`) into an npm-workspaces monorepo with `apps/app`, `apps/api`, `apps/web`, and `packages/shared`, replacing Prisma with Supabase.

**Architecture:** `apps/app` (UI, port 3000) proxies all `/api/*` calls to `apps/api` (Route Handlers only, port 3001) via Next.js rewrites. An `INTERNAL_API_KEY` header guards `apps/api`. Supabase `service_role` lives only in `apps/api`. Shared types, Zod schemas, and DB mappers live in `packages/shared`, consumed via source alias + `transpilePackages` (no dist build needed).

**Tech Stack:** Next.js 16.1.6, React 19.2.3, TypeScript 5.6, Supabase JS v2, Vitest 4, npm workspaces, concurrently, shadcn/ui, Tailwind CSS 4, Zod 4.

```bash
import { NextResponse, type NextRequest } from 'next/server';

export function middleware(req: NextRequest) {
  if (!req.nextUrl.pathname.startsWith('/api/')) {
    return NextResponse.next();
  }
  // Handle CORS preflight
  if (req.method === 'OPTIONS') {
    return new NextResponse(null, { status: 204 });
  }
  const key = req.headers.get('x-internal-key');
  if (!key || key !== process.env.INTERNAL_API_KEY) {
    return NextResponse.json(
      { data: null, error: { code: 'UNAUTHORIZED', message: 'Missing or invalid internal key' } },
      { status: 401 }
    );
  }
  return NextResponse.next();
}

export const config = {
  matcher: ['/api/:path*'],
};
```

```bash
cat > scripts/generate-seed.ts << 'EOF'
#!/usr/bin/env tsx
/**
 * Reads agents/agent-{n}-{stage}.md and generates supabase/seed.sql
 * No frontmatter required — stage derived from filename, name from first h1.
 */
import { readdirSync, readFileSync, writeFileSync } from 'node:fs';
import { join } from 'node:path';

const AGENTS_DIR = join(process.cwd(), 'agents');
const OUT = join(process.cwd(), 'supabase', 'seed.sql');
const AGENT_RE = /^agent-(\d+)-([a-z]+)\.md$/;

function dollarQuote(s: string): string {
  let tag = 'bt';
  let n = 0;
  while (s.includes(`$${tag}$`)) tag = `bt${++n}`;
  return `$${tag}$${s}$${tag}$`;
}

const files = readdirSync(AGENTS_DIR)
  .filter(f => AGENT_RE.test(f))
  .sort();

if (files.length === 0) {
  console.error('No agent files found in agents/ — expected agent-{n}-{stage}.md');
  process.exit(1);
}

const inserts = files.map(f => {
  const [, agentNum, stage] = AGENT_RE.exec(f)!;
  const id = `agent-${agentNum}-${stage}`;
  const slug = stage;
  const instructions = readFileSync(join(AGENTS_DIR, f), 'utf8').trim();
  const nameMatch = instructions.match(/^#\s+Agent\s+\d+:\s+(.+)/m);
  const name = nameMatch ? nameMatch[1].trim() : `Agent ${agentNum}`;

  return [
    `insert into public.agent_prompts (id, name, slug, stage, instructions, created_at, updated_at)`,
    `values (`,
    `  ${dollarQuote(id)},`,
    `  ${dollarQuote(name)},`,
    `  ${dollarQuote(slug)},`,
    `  ${dollarQuote(stage)},`,
    `  ${dollarQuote(instructions)},`,
    `  now(),`,
    `  now()`,
    `)`,
    `on conflict (slug) do update set`,
    `  name = excluded.name,`,
    `  instructions = excluded.instructions,`,
    `  updated_at = now();`,
  ].join('\n');
});

const header = [
  '-- GENERATED by scripts/generate-seed.ts — DO NOT EDIT BY HAND',
  '-- Source: agents/agent-*.md',
  '-- Regenerate: npm run db:seed',
  '',
].join('\n');

writeFileSync(OUT, header + '\n' + inserts.join('\n\n') + '\n');
console.log(`✓ Wrote ${inserts.length} agent_prompts inserts → supabase/seed.sql`);
EOF
```

```bash
cat > apps/api/src/lib/queries/discovery.ts << 'EOF'
import { createServiceClient } from '@/lib/supabase';
import { mapProjectFromDb, mapResearchArchiveFromDb } from '@brighttale/shared/mappers/db';
import { SupabaseError } from '@/lib/api/errors';

type DiscoveryInput = {
  research: {
    title: string;
    theme: string;
    research_content: string;
    sources?: Array<{ url: string; title: string; author?: string }>;
  };
  ideas: string[];
  defaults?: { goal?: string };
};

export async function createProjectsFromDiscovery(input: DiscoveryInput) {
  if (input.ideas.length === 0) {
    throw new Error('At least one idea must be selected');
  }
  const sb = createServiceClient();

  // 1. Create research archive
  const { data: research, error: researchErr } = await sb
    .from('research_archives')
    .insert({
      title: input.research.title,
      theme: input.research.theme,
      research_content: input.research.research_content,
    })
    .select()
    .single();

  if (researchErr || !research) throw new SupabaseError(researchErr ?? { message: 'Failed to create research' });

  // 2. Insert research sources if provided
  if (input.research.sources?.length) {
    const { error: srcErr } = await sb
      .from('research_sources')
      .insert(input.research.sources.map(s => ({ ...s, research_id: research.id })));
    if (srcErr) throw new SupabaseError(srcErr);
  }

  // 3. Create one project per idea
  const projectInserts = input.ideas.map(ideaId => ({
    title: `Project for ${ideaId}`,
    research_id: research.id,
    current_stage: 'brainstorm',
    completed_stages: [] as string[],
    auto_advance: true,
    status: 'active',
  }));

  const { data: projects, error: projErr } = await sb
    .from('projects')
    .insert(projectInserts)
    .select();

  if (projErr || !projects) throw new SupabaseError(projErr ?? { message: 'Failed to create projects' });

  return {
    research_id: research.id,
    project_ids: projects.map(p => p.id),
  };
}
```

```bash
export async function getTemplateResolved(id: string) {
  // Fetch template and its parent (one level) then merge config_json
  const sb = createServiceClient();
  const { data: template, error } = await sb
    .from('templates')
    .select('*, parent:parent_template_id(*)')
    .eq('id', id)
    .single();

  const parentConfig = JSON.parse((template.parent as { config_json: string }).config_json ?? '{}');

export async function listTemplates(type?: string) {
```

```bash
): Promise<ApiResult<T>> {

  return { data: json.data as T, error: null };
```

## Architecture

### Data Flow

Browser → `apps/app` (Next.js rewrite `/api/*` → `apps/api`) → Supabase (service_role)

All API calls from the UI go through the rewrite. `apps/app` injects `X-Internal-Key` header server-side. `apps/api/src/app/middleware.ts` validates it on every `/api/*` request.

### packages/shared

Consumed via source alias (`transpilePackages`). No need to run `build:shared` during dev.

After any schema change: run `npm run db:types` and commit the updated `packages/shared/src/types/database.ts`.

### Security

- `SUPABASE_SERVICE_ROLE_KEY` lives ONLY in `apps/api`.
- `INTERNAL_API_KEY` lives in both `apps/app` and `apps/api` (same value).
- RLS is enabled on all tables. Zero policies = deny-all for anon. `service_role` bypasses RLS.
- Auth (Supabase + `@tn-figueiredo/auth-nextjs`) is a future spec.

### DB Mappers

All snake_case ↔ camelCase conversion happens in `packages/shared/src/mappers/db.ts`. Query functions in `apps/api/src/lib/queries/` return camelCase. Never use snake_case column names outside of mapper functions.
