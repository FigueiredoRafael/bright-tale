# Live progress via Supabase Realtime, scoped per Project

Live Pipeline progress reaches the browser via Supabase Realtime subscriptions to the `stage_runs` and `job_events` tables, filtered by `project_id`. Each browser tab opens one channel per Project it cares about (`project:{id}`); RLS on both tables enforces ownership. Existing per-session SSE endpoints (`/api/research-sessions/:id/events`, etc.) stay during the transition but are deprecated. Alternatives considered: a custom SSE endpoint per Project (rejected — reinvents Realtime, more API surface, manual RLS-equivalent in code); long-polling (rejected — latency and DB cost). Realtime gives one subscription covering all Stages of a Project, survives Stage transitions and page reloads transparently, and scales naturally to dashboards listing many Projects (one channel per row).
