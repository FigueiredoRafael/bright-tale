# Plan: Key Takeaways — End-to-End Implementation

## Overview

Add a "Key Takeaways" summary box to blog posts. The Agent 3b-Blog generates
bullet points during content production, they flow through the pipeline, and
are published to WordPress as post meta.

**WordPress side is already done.** This plan covers only the bright-tale changes.

---

## WordPress Contract (already implemented)

- **Meta key:** `_bc_key_takeaways`
- **Format:** plain text, one bullet point per line, separated by `\n`
- **Example value:** `"AI reduces diagnostic errors by 85%\nAdoption is growing fastest in radiology\nRegulatory frameworks are still catching up"`
- **No markdown, no HTML, no hyphens/bullets** — just raw text per line
- **REST API:** `POST /wp-json/wp/v2/posts` with `"meta": { "_bc_key_takeaways": "line1\nline2\nline3" }`
- **If empty:** the summary box does not render on the frontend

### 3. Publish Route — `apps/api/src/routes/wordpress.ts`

After building `postData` (around line 1526), add meta injection:

```typescript
// --- existing code (line ~1526) ---
if (featured) postData.featured_media = featured.wpId;

// Key Takeaways → WordPress post meta
// Priority: explicit body param > draft JSON field
const keyTakeaways: string[] =
  body.keyTakeaways
  ?? (draftJson?.key_takeaways as string[] | undefined)
  ?? [];

if (keyTakeaways.length > 0) {
  postData.meta = {
    ...(postData.meta as Record<string, unknown> ?? {}),
    _bc_key_takeaways: keyTakeaways.join('\n'),
  };
}
```

This does two things:
- If `keyTakeaways` is explicitly passed in the publish request body, use that
- Otherwise, read `key_takeaways` from the draft JSON (generated by Agent 3b)
- Joins the array into newline-separated text (the format WordPress expects)
