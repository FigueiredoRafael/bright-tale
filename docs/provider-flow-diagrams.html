1. Step by Step how it happens over time

How a setting saved by the admin ends up showing as a recommendation on screen and being used when content runs.

sequenceDiagram actor Admin participant DB as Settings Database participant Cache as Settings Loader participant Wizard as Autopilot Wizard participant Engine as Content Engine participant MP as AI Picker (on screen) participant Job as Background Job participant RPO as AI Decision Logic Admin->>DB: Save preferred AI, model, and tools Admin->>DB: Toggle provider on or off, update models list Note over Wizard: User opens autopilot setup Wizard->>DB: Fetch active providers and their models DB-->>Wizard: Returns active list with model choices Wizard->>DB: Fetch each agent's recommended AI and model DB-->>Wizard: Returns recommendations per stage Wizard->>Wizard: Pre-fill each stage with the recommendation Note over Wizard: User can override any stage before starting Note over Engine: Page opens — loads saved AI settings Engine->>+Cache: Ask for saved settings Cache->>DB: Look up preferred AI and model for this agent DB-->>Cache: Returns the saved row Cache-->>-Engine: Here is the preferred AI and model Engine->>Engine: Apply autopilot override if one was set in wizard Engine->>Engine: Fall back to agent recommendation if no override Engine->>MP: Show the active choice as recommended MP-->>MP: Displays ✦ Recommended badge Note over Engine: User clicks Generate (or autopilot runs) Engine->>+Job: Start job with chosen AI, model, and per-stage overrides Job->>+Cache: Load saved settings Cache->>DB: Check which providers are currently active DB-->>Cache: Returns active provider list (cached 60 seconds) Cache-->>-Job: Here are the settings — skip disabled providers Job->>+RPO: Which AI should we actually use? Note right of RPO: User override beats agent default beats system default Note right of RPO: Disabled providers are skipped entirely RPO-->>-Job: Use this AI and model Job->>Job: Run AI, fall back to next provider if it fails Job-->>Engine: Live progress updates
