O que muda tecnicamente:

1. Adicionar tokens Tailwind faltantes

Definir dash-card, dash-border, v-primary, v-secondary, e cores vivid-* no globals.css do web app para que os componentes do @tn-figueiredo/admin renderizem corretamente.

2. Garantir classe dark no HTML

O AdminShell usa dark: variants do Tailwind. O <html> precisa ter a classe dark.

3. Refatorar dashboard page

Trocar inline styles por Tailwind classes. Usar os componentes KpiSection e KpiCard corretamente com os tokens definidos. Grid 2 colunas no desktop.

4. Isolar CSS admin vs landing page

O globals.css atual mistura tokens de landing page com o admin. Garantir que o admin scope tenha seus próprios valores sem conflitar.
