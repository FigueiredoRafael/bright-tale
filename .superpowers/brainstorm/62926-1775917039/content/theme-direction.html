Qual direção de tema para o Admin?

O admin atual herda o tema dark da landing page. Precisamos de um tema próprio para o admin.

A

Light Theme (estilo TôNaGarantia)

Fundo branco/cinza claro, cards brancos com bordas sutis, sidebar branca, tipografia escura. Exatamente como o dashboard do TôNaGarantia na Image #2. Usar os tokens de apps/app/globals.css que já existem (--background: #F7F9FC, --card: #FFFFFF, etc.).

Pros

  • Tokens já existem no projeto (apps/app)
  • Referência visual clara (TôNaGarantia)
  • Mais legível para dashboards com muitos dados
  • Consistente com shadcn/ui defaults

Cons

  • Precisa isolar CSS do admin vs landing page
B

Dark Theme Polished

Manter dark mas corrigir: ajustar contraste, bordas, espaçamento, tipografia. Usar os tokens dark da landing page mas aplicados com mais cuidado no admin.

Pros

  • Menos mudança — mesma paleta
  • Visualmente "moderno" para alguns

Cons

  • Difícil igualar qualidade da referência (TôNaGarantia é light)
  • Dark dashboards precisam de muito mais cuidado com contraste
  • A referência visual (Image #2) é light — difícil traduzir
C

Light + Dark Toggle

Implementar ambos os temas com toggle (o AdminShell já tem features: { darkMode: true }). Light como default, dark como opção.

Pros

  • Máxima flexibilidade
  • AdminShell já suporta dark mode toggle

Cons

  • Mais trabalho — precisa manter 2 conjuntos de tokens
  • Escopo maior para o que é essencialmente um fix de UI

Minha recomendação: Opção A (Light Theme). A referência é light, os tokens já existem em apps/app, e é o caminho mais direto para chegar na qualidade da Image #2. O admin precisa de CSS isolado da landing page — isso é uma mudança no globals.css do web app para o escopo /admin.
