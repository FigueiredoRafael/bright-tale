Content Core Agent — what changes
New input field: persona_context
persona_context: {
  name: "Leo Marcos",
  domain_lens: "Every business is
    a system...",
  analytical_lens: "Failure mode →
    fix → cost",
  strong_opinions: [...],
  approved_categories: [...]
}
New rule injected into Content Core
"Frame the thesis and argument chain through this persona's analytical lens. The thesis must reflect how [name] would interpret this evidence. Use their strong_opinions as the editorial position where the research supports it. Reject angles outside approved_categories."
Result: Same research → "FIRE Opportunity Cost" becomes a systems-architecture argument when Leo writes it, not a personal finance explainer.
Blog Agent — what changes
New input field: persona
persona: {
  name, bio_short,
  writing_voice_json: {
    writing_style,
    signature_phrases,
    characteristic_opinions
  },
  soul_json: {
    humor_style,
    recurring_jokes,
    language_guardrails
  }
}
New rule injected into Blog Agent
"Write this post as [name]. Use their writing_style for tone. Drop a signature_phrase where it fits naturally — never forced. Express characteristic_opinions as conclusions the evidence leads to, not as rants. Apply humor_style sparingly. Follow language_guardrails as hard rules."
Global guardrails (12 AI vices) stay in Blog Agent's base rules.content array. Persona guardrails append on top at call time. No duplication.
